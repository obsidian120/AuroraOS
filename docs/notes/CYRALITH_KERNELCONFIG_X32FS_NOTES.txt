Cyralith Kernelconfig + x32FS Console Build

Aenderungen:
- 'settings' wurde in der Benutzeroberflaeche zu 'kernelconfig' umbenannt.
  Der alte Befehl 'settings' bleibt als Kompatibilitaets-Alias erhalten und weist auf kernelconfig hin.
- Kernelconfig ist optisch jetzt menuconfig-aehnlicher: Kategorien, [*]/[ ]-Toggles, < >-Eintraege, Statuszeile.
- Neue FS-Kategorie in der Kernelconfig:
  kernelconfig fs
  Optionen: FS-Persistenz, Autosave, strenge FS-Rechte, Systempfad-Schutz,
  x32FS Console, Recovery-Scan, Save/Load.
- Neue native CyralithFS-Konsole:
  x32fs
  fsconsole
  app run x32fs
  oder ueber den Desktop-Eintrag 'x32 FS Console'.
- Die x32FS Console nutzt CyralithFS direkt und ist bewusst getrennt von der normalen Shell.
- Alle frueheren SSF-Namen im OS-Code wurden zu CCS umbenannt.
  CCS steht hier als Cyralith Core Standard/Alignment-Platzhalter.
- SSF/CCS-artige Kernelconfig-Logik liegt getrennt in:
  kernel/src/ccs_settings.h
  kernel/src/ccs_settings.c

Wichtige Befehle:
  kernelconfig
  kernelconfig fs
  kernelconfig ccs
  kernelconfig display
  sysdo cyralith kernelconfig
  x32fs
  app run x32fs
  open desktop

Build:
  cd cyr_io/cyralith/kernel
  make clean
  make all
  make run

Hinweis:
- 'make all' wurde erfolgreich geprueft.
- 'make iso/run' haengt weiterhin davon ab, ob grub-mkrescue/qemu auf dem Host installiert und berechtigt ist.
