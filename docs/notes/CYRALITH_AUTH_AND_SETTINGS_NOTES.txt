Cyralith settings/auth cleanup
================================

- The old sudo-like command is now named `sysdo`.
  Example: `sysdo cyralith settings`

- `settings` still opens the full kernel/system settings UI.
  `settings kernel` is only a direct jump into the kernel-protection page.
  `kernelconfig ccs` is only a direct jump into the CCS page.

- CCS settings now have their own kernel module files:
  kernel/src/ccs_settings.h
  kernel/src/ccs_settings.c

- The CCS values are still persisted through the existing kernel settings snapshot,
  but the code boundary is prepared so CCS can later become its own policy block.

- Desktop settings are intentionally not mixed into shell settings.
